# マルチモーダルPIIフィルタ設計

## 目的

画像、PDF、動画に含まれる個人情報を扱うための段階的な設計である。現在のCloakroomはテキストだけをマスクし、画像バイト列やbase64データをOCRサービスへ自動送信しない。

## 守る境界

- 元画像とOCR結果はローカルで処理する。
- 外部OCRや外部の顔検出サービスへ無断送信しない。
- 検出に失敗したメディアを「安全」とは扱わず、利用者へ未検査であることを伝える。
- テキスト化した結果は既存の `PIIFilter` と同じ辞書、正規表現、プラグイン経路を通す。

## Phase 1: OCRテキスト

1. `image` / PDF入力を明示的な設定で受け付ける。
2. ローカルOCRアダプタがテキストと位置情報を返す。
3. テキストを既存のフィルタへ渡し、検出結果を位置情報へ戻す。
4. 元画像には検出領域を塗りつぶしてから上流へ送る。

OCRエンジンはオプション依存として分離する。Cloakroom本体のゼロランタイム依存方針を保つため、Tesseract.jsなどは導入を選んだ利用者だけが追加する。

## Phase 2: 顔と身分証

- 顔検出はローカルモデルで矩形だけを返し、矩形をモザイクまたは塗りつぶしする。
- 運転免許証・マイナンバーカードはレイアウト検出後、番号と顔を個別にマスクする。
- 自動判定の信頼度が低い場合は、送信を止めるか未検査として扱う設定を提供する。

## Phase 3: 動画

動画はフレーム抽出、Phase 1/2の検出、時間的に連続する追跡、再エンコードの順で処理する。処理負荷が高いため、まず静止画・PDFで精度と誤検出時の操作を確立してから着手する。

## API方針

メディアごとの検出器は、テキストPIIと同じカテゴリ・信頼度・座標を返す。プロキシ本体は特定のOCR/顔検出ライブラリを直接参照せず、アダプタを通して機能を追加する。
